# Sprint 05 - Compatibility Panel and Corpus Tests

Parent: [[Sprint Cadence and Capacity]]
Dates: May 11, 2026 to May 24, 2026

## Sprint Goal
Expose compatibility transparency in UI and stabilize corpus-driven validation pipelines.

## Commitments
- Epic 06 primary.
- Epic 01 and 02 status feed integration.
- Epic 07 dashboard depth increase.

## Stories
1. Build compatibility issue schema shared by UI and CLI.
2. Implement panel with workbook/sheet issue drill-down.
3. Integrate parser and calc capability detectors.
4. Expand corpus and nightly no-repair + diff checks.
5. Add dashboard for compatibility trend tracking.

## Acceptance Criteria
- Users can see supported/partial/preserved/not supported statuses.
- CLI and UI compatibility outputs match schema and content.
- Corpus nightly publishes pass/fail and regression artifact bundle.

## Exit Signals
- >= 80% no-repair rate on active corpus slice.
