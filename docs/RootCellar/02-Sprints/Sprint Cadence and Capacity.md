# Sprint Cadence and Capacity

Parent: [[docs/RootCellar/RootCellar Planning Hub]]
Roadmap: [[docs/RootCellar/00-Program/Milestone Roadmap]]

## Cadence
- Sprint length: 2 weeks.
- Start day: Monday.
- Ceremony rhythm:
  - Monday: planning + dependency sync.
  - Wednesday week 1: architecture + security checkpoint.
  - Wednesday week 2: backlog freeze for next sprint.
  - Friday week 2: demo + retro + metric review.

## Team Capacity Model (Planning Baseline)
- Engine team: 24 points/sprint.
- UI team: 20 points/sprint.
- Scripting/Security team: 18 points/sprint.
- DevEx/CLI/Quality team: 18 points/sprint.
- Shared bug/interrupt budget: 15% capacity reserved.

## Exit Metrics Per Sprint
- Planned vs completed story points.
- Carryover percentage.
- Corpus no-repair pass delta.
- Golden formula pass delta.
- p95 UI and recalc latency deltas.
- Observability completeness score.

## Working Agreements
- No uninstrumented feature merges for critical paths.
- All stories include acceptance tests.
- Security-relevant changes require threat note update.
